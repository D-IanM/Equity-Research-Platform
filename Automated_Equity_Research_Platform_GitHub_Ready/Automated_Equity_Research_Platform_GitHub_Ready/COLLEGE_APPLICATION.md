## Resume version

**Automated Equity Research Platform — Founder and Lead Developer**

- Designed a modular Google Colab and Python platform for market-data collection, financial statement normalization, ratio analysis, DCF and comparable valuation, risk measurement, and constrained portfolio optimization.
- Implemented reproducible data pipelines, validation, retries, error handling, unit tests, interactive Plotly visualizations, and automated Markdown/HTML equity-research reports.

## Interview talking points

Be prepared to explain:

1. Why the project uses separate data, model, and presentation layers.
2. Why a DCF is sensitive to discount rate and terminal growth.
3. Why sample mean returns make portfolio optimization unstable.
4. How statement classifications differ across companies.
5. Why the report uses a research signal rather than claiming certainty.
6. One bug, data-quality issue, or model limitation you found and corrected.
7. Which parts you personally modified after reviewing the generated foundation.

## Integrity note

Do not claim that you independently authored code you did not review. Run the notebooks, test edge cases, change assumptions, inspect failures, add your own company thesis, and keep a development log. The strongest application story is not that the code exists; it is that you can explain how you validated and improved it.
